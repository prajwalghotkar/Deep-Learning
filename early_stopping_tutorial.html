01 · Concept

What is Overfitting?

Imagine a student who, before an exam, only memorizes previous years' papers — every question, every answer by rote. But when new questions appear, they fail completely. That is overfitting.

A model learns the training data so thoroughly that it memorizes its patterns — including noise and anomalies. On new, unseen data, this model simply does not work.

Overfitting occurs when a model performs very well on training data but very poorly on validation/test data. Training loss keeps falling, but validation loss starts rising after a certain point.

02 · Solution

Why is Early Stopping Necessary?

If we train a model for too many epochs — say 3500 — it gradually drifts into overfitting territory. The simplest solution is to stop training at the right moment.

Early Stopping is a callback that monitors validation loss (or any other metric) during training. When improvement stalls, training halts automatically.

Think of a wise coach who says — "If you haven't learned anything new in the last 20 sessions, stop drilling and go practice." That is exactly what Early Stopping does for a model.

03 · Code

EarlyStopping in Keras

Using Early Stopping in TensorFlow/Keras is straightforward — just define a callback and pass it into model.fit():

from tensorflow.keras.callbacks import EarlyStopping

# Step 1: Define the callback
callback = EarlyStopping(
    monitor    = "val_loss",
    min_delta  = 0.00001,
    patience   = 20,
    verbose    = 1,
    mode       = "auto",
    baseline   = None,
    restore_best_weights = False
)

# Step 2: Pass it into training
history = model.fit(
    X_train, y_train,
    validation_data = (X_test, y_test),
    epochs          = 3500,
    callbacks       = callback   # ← right here
)

The model will not run all 3500 epochs — as soon as validation loss stops improving (based on patience), training will stop automatically.

04 · Parameters

Every Parameter Explained

EarlyStopping has 7 parameters. Let's go through each one:

monitor
What to watch? Default: "val_loss". Specifies which value to track during training. Common options: "val_loss", "val_accuracy", "loss".
min_delta
How much improvement is enough? If improvement is less than min_delta, it counts as "no improvement." Example: 0.00001 means even a tiny 0.001% gain is not considered progress.
patience
How many epochs to wait? If there is no improvement for patience consecutive epochs, training stops. patience=20 means wait for 20 epochs before giving up.
verbose
Print logs? 0 = silent, 1 = print a message when training stops. Keep it at 1 during development.
mode
Direction of improvement? "min" = for loss (should decrease), "max" = for accuracy (should increase), "auto" = Keras infers it automatically.
baseline
Minimum standard to beat? If set, the model must first reach this value before Early Stopping activates. Set to None for no baseline requirement.
restore_best_weights
Roll back to the best state? When True, the model reverts to the weights from its best epoch. In production, this should always be True!

In practice: patience=10–50, restore_best_weights=True, and monitor="val_loss" is a safe, effective combination for most problems.

06 · Summary

Summary

Here is what we covered today:

Overfitting
The model memorizes training data and fails on new, unseen data.
Early Stopping
Monitors validation loss and stops training at the right moment.
patience
How many epochs to wait before stopping when no improvement is seen.
min_delta
How small an improvement counts as "no change."
restore_best_weights
Always set to True in production — restores the model's best state.
# Production-ready Early Stopping
callback = EarlyStopping(
    monitor              = "val_loss",
    patience             = 20,
    min_delta            = 0.0001,
    mode                 = "auto",
    restore_best_weights = True,  # ← always include this!
    verbose              = 1
)
